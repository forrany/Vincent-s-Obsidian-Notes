Vue Core > Reactivity System

核心定义

Vue 的响应式系统是通过数据劫持+依赖收集实现的自动更新机制，当数据变化时能精确更新相关视图。

工作原理

关键过程

  1. 初始化阶段：

    // 示例：reactive() 实现
function reactive(obj) {
  return new Proxy(obj, {
    get(target, key) {
      track(target, key) // 依赖收集
      return Reflect.get(...arguments)
    },
    set(target, key, value) {
      trigger(target, key) // 触发更新
      return Reflect.set(...arguments)
    }
  })
}

  2. 依赖收集：通过 effect 建立数据与组件的映射关系

  3. 派发更新：采用异步队列 (nextTick) 批量更新

版本差异

特性 Vue 2 (defineProperty) Vue 3 (Proxy)
检测范围 仅对象属性 全对象操作
数组处理 需要hack覆盖方法 原生支持
性能 中等 更优

典型应用

  1. 组件状态：

    <script setup>
const state = reactive({ count: 0 })
</script>

  2. 计算属性（Vue Computed）：

    const double = computed(() => state.count * 2)

  3. 副作用（Vue Watch）：

    watch(() => state.count, (newVal) => {
  console.log('Count changed:', newVal)
})

深度优化

  1. 响应式转换控制：
    • shallowReactive() 浅层响应
    • markRaw() 跳过代理
  2. 性能陷阱：
    • 避免超大响应式对象（性能优化指南）
    • 合理使用 effectScope（RFC-0041）

关联原理

  • Virtual DOM（更新触发路径）
  • JavaScript Proxies（底层机制）
  • Reactive Programming（设计范式）

常见误区

❌ 直接解构会丢失响应性：

const { count } = state // 非响应式

✅ 正确方式：

const count = toRef(state, 'count')

实践验证

<template>
  <button @click="state.count++">
    {{ state.count }} ({{ double }})
  </button>
</template>

import { reactive, computed } from 'vue'
const state = reactive({ count: 0 })
const double = computed(() => state.count * 2)
</script>
